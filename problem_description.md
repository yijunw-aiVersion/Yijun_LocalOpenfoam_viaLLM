# 技术面试项目：自然语言驱动的CFD仿真工作流构建器

## 项目背景

我们希望你在一周内完成一个产品原型：一个面向工程师和研究人员的自然语言驱动的CFD仿真系统。

这个产品的目标是让用户不需要手动编写OpenFOAM配置文件、不需要逐步执行命令行操作、不需要处理复杂的前后处理流程，只通过自然语言描述仿真参数，就能让系统自动完成从网格生成到结果可视化的完整CFD仿真流程。

你可以把它理解为：用户说出仿真需求，产品负责**理解、配置、执行并呈现结果**。



## 项目目标

请设计并实现一个产品，让用户能够通过自然语言输入参数来完成圆柱绕流CFD仿真。

具体来说，用户应该能够通过自然语言描述以下参数：



> "圆柱直径0\.1米，雷诺数100，来流速度1米每秒。"
> 
> 
> 
> "我想模拟一个直径5厘米的圆柱，流速2m/s，雷诺数200。"
> 
> 
> 
> "圆柱半径0\.05m，Re=150，U=1\.5m/s，流体是水。"
> 
> 
> 
> "模拟空气绕过直径10cm的圆柱，速度3m/s。"
> 
> 

这些只是示例。实际评测时，我们会用不同的表述方式和参数组合来测试你的产品。

你的产品需要**\*\*准确理解\*\***用户输入的参数，并自动完成完整的仿真流程。



## 交付结果

1. 一个可以运行的产品原型，产品形式不限，可以是 Web App、CLI、桌面应用或其他交互形式；

2. Private Github Repo Read 权限，用于验证评价开发历史和代码质量；

3. 简单的运行说明，包括环境配置要求（如OpenFOAM版本）；

4. AI Coding 工具使用记录，例如Cursor/Claude Code等工具的使用历史导出。

## 产品需要体现的核心能力

我们不会规定你必须怎么实现，也不会规定具体技术架构。你需要自己判断如何设计。

但产品最终需要体现以下能力：

### 1\. 能理解自然语言输入的仿真参数

用户可能用各种不同的方式描述参数，例如：

- "圆柱直径0\.1米" vs "圆柱半径5厘米"

- "雷诺数100" vs "Re=100"  

- "流速1m/s" vs "来流速度1米每秒"

- 可能同时给出雷诺数和流速，也可能只给其中一个

产品需要能够**\*\*正确解析\*\***这些不同的表述方式，识别参数值和单位，并在必要时通过追问补全缺失信息。



### 2\. 能自动配置OpenFOAM仿真案例

产品需要根据解析出的参数，自动生成完整的OpenFOAM算例，包括：

- 几何建模（圆柱绕流的二维或三维几何）

- 网格划分（blockMesh或snappyHexMesh等）

- 边界条件设置（根据雷诺数和流速计算流体属性）

- 求解器配置（选择合适的求解器如simpleFoam或pimpleFoam）

- 计算参数设置（时间步长、收敛标准等）

### 3\. 能自动执行仿真并监控进度

产品需要能够：

- 调用OpenFOAM命令行工具执行仿真

- 实时显示仿真进度

- 检测收敛状态



### 4\. 能生成并呈现仿真结果

仿真完成后，产品需要自动提取并可视化以下结果：

**稳态流速场**：整个计算域的速度分布云图

**圆柱表面压力场**：圆柱表面的压力分布曲线或云图

可视化方式不限，可以是：

- ParaView脚本生成图片

- Python后处理（matplotlib等）

- 交互式Web可视化

- 其他合适的方式
